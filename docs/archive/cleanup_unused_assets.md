# 不要ファイル・不要コードの削除プロポーサル

## 概要
プロジェクトのメンテナンス性を向上させるため、不要と判断されたファイルおよびコードの削除・アーカイブを提案します。

## 1. 削除対象ファイル (アーカイブ推奨)

以下のファイルは、リファクタリング前の検証用スクリプト、または既に機能が他のモジュールに統合された古いエントリーポイントです。これらは現在使用されておらず（一部は依存ファイルが欠損しており動作しません）、削除またはアーカイブディレクトリへの移動を推奨します。

| ファイル名              | 判定理由                                                                                                           |
| :---------------------- | :----------------------------------------------------------------------------------------------------------------- |
| `verify_strategies.py`  | 存在しない `stock_analyzer.py` を呼び出しており、動作不能な古い検証スクリプトです。                                |
| `verify_refactoring.py` | 手動でのストラテジー検証を行う一時的なスクリプトであり、現在は `tests/` 以下の正規テストがその役割を担っています。 |
| `collector.py`          | データ収集ロジックは `src/commands/ingest.py` (IngestCommand) に統合されており、冗長です。                         |
| `src/calc/v2.py`        | クラス定義内で `DeprecationWarning` を送出するだけのラッパーメソッドが残されており、実質的にデッドコードです。     |

## 2. 削除対象コード (特定ファイル内)

### `src/calc/v2.py`
*   **内容:** `V2ScoringMixin` クラスおよびそのメソッド群。
*   **理由:** `ScoringEngine` への移行が完了しており、後方互換性のために残されていましたが、カバレッジ低下の要因にもなっており、削除しても現行の `src/calc/engine.py` を使用するコードには影響しません。
*   **処置:** ファイルごと削除し、もし参照している箇所があれば `src.calc.engine` から直接インポートするように修正します。

## 3. アクションプラン

1.  **アーカイブ移動:** 上記ファイルを `archive/` ディレクトリへ移動（または削除）。
2.  **依存確認:** `src/calc/v2.py` を削除した後、`self_diagnostic.py` を実行して影響がないことを確認。
3.  **コミット:** 変更履歴 (`history/`) を更新し、クリーンアップな状態を保存。
