## [15:37] No.1: API Rate Limit Error (Parallel Execution)
- **概要**: `stock_analyzer.py --workers 4` 実行時に `Too Many Requests` エラーが多発する。
- **原因**: Yahoo Finance API に対する並列リクエスト数が過多であり、レートリミットに抵触している。
- **影響**: データ取得が失敗し、分析が進行しない。並列数を上げるほど悪化する可能性がある。
- **修正案**:
    1. `src/data_fetcher.py` 内のリクエスト間隔 (sleep) を調整する。
    2. 並列実行時のワーカーごとにランダムな待機時間を設ける。
    3. 同時接続数 (`--workers`) を制限する運用ルールを設ける。
